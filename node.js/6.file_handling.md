# ðŸ“‚ File Handling in Node.js

## ðŸ“˜ Introduction

File handling in Node.js is done using the **`fs` (File System)** module.
It allows you to **create**, **read**, **update**, **delete**, and **rename** files on your computer.

To use it:

```js
const fs = require('fs');
```

---

## ðŸ§© Types of File Operations

There are two main types of file operations in Node.js:

| Type                            | Description                                                      |
| ------------------------------- | ---------------------------------------------------------------- |
| **Synchronous (Blocking)**      | Operations block the execution until completed.                  |
| **Asynchronous (Non-Blocking)** | Operations execute in the background with callbacks or promises. |

---

## ðŸª„ Common File Operations

### 1. **Creating / Writing Files**

#### ðŸŸ¢ Asynchronous:

```js
fs.writeFile('example.txt', 'Hello Node.js!', (err) => {
  if (err) throw err;
  console.log('File created and data written!');
});
```

#### ðŸŸ£ Synchronous:

```js
fs.writeFileSync('example.txt', 'Hello Node.js!');
console.log('File created and data written!');
```

> ðŸ“ If the file exists, it will **overwrite** the content.

---

### 2. **Appending Data to a File**

#### Async:

```js
fs.appendFile('example.txt', '\nNew line added.', (err) => {
  if (err) throw err;
  console.log('Data appended!');
});
```

#### Sync:

```js
fs.appendFileSync('example.txt', '\nNew line added.');
```

---

### 3. **Reading Files**

#### Async:

```js
fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});
```

#### Sync:

```js
const data = fs.readFileSync('example.txt', 'utf8');
console.log(data);
```

> Always specify `'utf8'` to read data as text.

---

### 4. **Renaming Files**

```js
fs.rename('example.txt', 'newExample.txt', (err) => {
  if (err) throw err;
  console.log('File renamed successfully!');
});
```

---

### 5. **Deleting Files**

```js
fs.unlink('newExample.txt', (err) => {
  if (err) throw err;
  console.log('File deleted!');
});
```

---

### 6. **Checking if a File Exists**

```js
fs.access('example.txt', fs.constants.F_OK, (err) => {
  console.log(err ? 'File does not exist' : 'File exists');
});
```

---

### 7. **Working with Directories**

| Operation     | Method         | Example                                  |
| ------------- | -------------- | ---------------------------------------- |
| Create folder | `fs.mkdir()`   | `fs.mkdir('test', err => {...})`         |
| Read folder   | `fs.readdir()` | `fs.readdir('.', (err, files) => {...})` |
| Delete folder | `fs.rmdir()`   | `fs.rmdir('test', err => {...})`         |

## âš¡ Using Promises (Modern Way)

You can also use the **Promise-based API** from `fs.promises` for cleaner code:

```js
const fs = require('fs').promises;

async function readFileExample() {
  try {
    const data = await fs.readFile('example.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

readFileExample();
```

---

## ðŸ§  Key Points

* Always handle **errors** in file operations.
* Prefer **asynchronous methods** to avoid blocking.
* Use **`fs.promises`** or **`async/await`** for cleaner asynchronous code.
* Node.js file operations are performed relative to the **current working directory**.
